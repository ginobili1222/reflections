查看文件的两个版本之间的差异如何帮助你了解引入的错误？
通过比较出现bug的新版本与可以正常运行的较早版本,找出两者之间的不同之处，那么可以确定造成bug出现的原因一定因为对较早版本的改动而造成的，从而缩小查找错误的范围，提高效率。

从长远来看，能轻松访问文件的整个历史记录将如何提高编程者的效率？
可以让编程者在旧版本的基础上进行开发，同时当新版本出现bug时，可以通过与旧版本的比较迅速确定造成bug的原因。

你认为，手动选择何时创建提交（像在 Git 中做的那样）与自动保存版本（如 Google Docs 所做的那样）各有何优缺点？
手动创建提交可以在你主观逻辑判断合适时进行，因此手动创建提交的版本更具有逻辑性，并且版本的改动不会太大也不会太小；而自动保存的版本则可以防止一些意外的发生，如宕机、忘记保存等情况。

为何一些版本控制系统（例如 Git）允许在一次提交中保存多个文件，而其他系统（例如 Google Docs）单独对待每个文件？
因为有些各个文件中存在很强的关联性，而有些则是各个文件相对独立。

如何使用命令 git log 和 git diff 来查看文件的历史记录？
首先进入需要查看文件的目录，通过git log查看文件历史版本的记录，可以获得各个版本的id，使用git diff加需要比较的两个版本的id号来查看不同版本之间的不同之处。

使用版本控制，如何让你更有信心进行有可能会造成破坏的更改？
因为使用git checkout可以使程序回归至以前的版本，从而我可以很轻松的找到bug第一次出现的版本，从而使用git diff命令，通过对比找到造成bug的原因。
